// The Doctrine

Keep It Simple.
Keep It Deadly.

Oedon is not a product — it is a philosophy applied to infrastructure. Every tool must earn its place: no web panels, no management databases, no interfaces that obscure what is actually happening. If you can't audit it in a terminal, it doesn't belong in the stack.

Born from frustration with cloud lock-in and bloated PaaS platforms, Oedon is a portable fortress — a folder you carry to any VPS, run docker compose up, and own a hardened server.

Invisible but omnipresent. It doesn't appear in top. It doesn't consume. It protects and serves with the maximum efficiency of pure Nginx.

— The Oedon Doctrine

On the name: Oedon is a Great One from Bloodborne — a god without form, invisible and omnipresent, existing only in the unseen. A fitting namesake for infrastructure that should be felt but never seen: no processes to expose, no web surfaces to attack, no form for adversaries to grip.

Every piece earns its place.

No NPM. No Nginx Proxy Manager. No Netdata. Each component is the smallest, fastest, most auditable tool for the job.

Official Alpine image. Configured via envsubst templates — no database, no interface. Three vhosts: static, wordpress, python. If it starts, it works.

Runtime

Docker Compose

The entire platform is a single docker compose up. A shared oedon-network connects all services while keeping them isolated from the host.

🔒
Custom SPA Auth

Portero Digital

Custom UDP knock daemon — HMAC-SHA256 + timestamp validation. Port 2222 opens for 60 seconds, then locks. Built in Python, zero external dependencies.

Intrusion Prevention

Fail2ban

Custom filters for WordPress wp-login.php (5 fails → 1h ban) and xmlrpc.php (2 hits → 24h ban). Ban action writes directly to ufw.

Certbot Container

Stays stopped — exit 0, zero resources. Run ./scripts/oedon-certbot.sh obtain domain.com when you need real certificates. Nothing runs unless summoned.

Monitoring

Custom terminal dashboard. ASCII banner, CPU/RAM/disk bars, Docker container status, top processes, failed SSH attempts. Runs as MOTD on every login.

🐍
Notary Service

Flask app tracking deployment integrity via /sign and /verify endpoints. If a container's code changes without authorization, Oedon knows.

📦
Backup

99-backup.sh archives SSH config, Fail2ban, UFW state and Docker volumes. A single curl sends it to Telegram. Up to 2 GB, free, on your phone.

// Custom Security Tool

Portero Digital.
Built from scratch.

Instead of relying on external tools, Oedon ships its own UDP knock daemon: portero.py. It listens on UDP port 62201, validates a HMAC-SHA256 payload with timestamp tolerance, and uses ufw to open port 2222 for exactly 60 seconds.

The client — knock.py — sends a single UDP datagram with a timestamp and HMAC signature. Old timestamps are rejected, making replay attacks impossible. The server runs as a systemd service installed by install.sh.

✓ HMAC-SHA256 + timestamp validation
✓ Replay attack protection (30s tolerance)
✓ Auto-close after 60 seconds
✓ Systemd service — starts on boot
✓ Zero external dependencies — stdlib only
✓ Port 2222 not open in UFW by default
portero.py — systemd
# From your local machine: $ python3 knock.py 192.168.1.141 Knock sent to 192.168.1.141:62201   # Server log (portero.py): [PORTERO] Knock from 192.168.1.50 [PORTERO] HMAC valid — port 2222 open (60s)   $ ssh m-kamil@oedon-server -p 2222 Welcome to OEDON ██████╗   [PORTERO] 60s elapsed — port 2222 closed [PORTERO] Active session unaffected.

// Defence in Depth

The Fortress Layers

L1

Portero Digital — Custom SPA

SSH port 2222 closed by default in UFW. Opens only after a valid HMAC-SHA256 UDP packet from knock.py. Closes automatically after 60s.

CUSTOM
L2

UFW Firewall

Default deny incoming. Only ports 80 and 443 face the world. Port 81, 19999 — not installed. Management is terminal-only via SSH.

ACTIVE
L3

Fail2ban — WordPress Hardened

Custom filters: wordpress-login.conf (5 fails → 1h ban) and wordpress-xmlrpc.conf (2 hits → 24h ban). Ban action: ufw.

ACTIVE
L4

Docker Network Isolation

All containers on oedon-network (172.18.0.0/16). Internal services expose no ports to the host. Only oedon-proxy faces the world.

ACTIVE
L5

SSH Hardening

Custom port 2222. sshd_config in version control. PrintMotd no — PAM handles MOTD via oedon-stats.sh on every session.

ACTIVE
L6

TLS Everywhere

Self-signed wildcard cert for *.oedon.test in development. Certbot container ready for Let's Encrypt on production. HTTP → HTTPS on all vhosts.

ACTIVE

// Infrastructure as Code

One folder.
Any VPS.
Full fortress.

Clone the repo, configure .env, run. No setup wizards. No web screens. No state outside the folder. Scripts are numbered — the filename is the documentation.

Every config lives in version control: Nginx vhosts, Fail2ban jails, SSH config, Docker Compose. CI/CD validates structure on every push via GitHub Actions.

// Autonomous Guardian

oedon-watchdog.sh

Every 5 minutes. 30-minute cooldown to prevent spam. Reads thresholds from .env. Silent when calm. Loud when needed.

💾

Disk Usage

Alerts before the disk fills. Threshold: WATCHDOG_DISK_THRESHOLD in .env.

every 5 min
📊

RAM & Load

Memory pressure and CPU load vs. nproc. Thresholds configurable.

every 5 min
🐳

Container Health

Checks all containers in WATCHDOG_CONTAINERS. Detects crashed or restarting.

every 5 min
⚔️

SSH Intrusion

Counts failed auth from /var/log/auth.log. Alert if >10 attempts in 5 min.

every 5 min
